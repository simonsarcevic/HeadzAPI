# HeadzAPI

A lightweight and efficient API for Paper servers to create and manage custom player heads using textures. It includes a simple builder pattern and an integrated caching system to optimize performance.

## Features

- **Fluent Skull Builder:** Easily construct custom skull `ItemStack`s with a clean builder pattern.
- **Texture Support:** Apply any custom head texture using its Base64 value.
- **Built-in Caching:** Automatically caches generated skulls to minimize performance impact.
- **Automatic Cache Expiration:** Stale cache entries are periodically cleaned up to manage memory, with a default Time-To-Live (TTL) of 10 minutes.

## How to Use

HeadzAPI provides a `SkullBuilder` for creating new heads and a `SkullCache` for managing them efficiently. It is recommended to always check the cache before creating a new skull to improve performance.

### Creating a Custom Skull

Here is an example of how to create a custom head, utilizing the cache to avoid recreating the `ItemStack` unnecessarily.

```java
import de.verdox.headzAPI.builder.SkullBuilder;
import de.verdox.headzAPI.cache.SkullCache;
import org.bukkit.entity.Player;
import org.bukkit.inventory.ItemStack;

public class SkullExample {

    public void giveCustomHead(Player player) {
        // Base64 texture value for the custom head
        String texture = "eyJ0ZXh0dXJlcyI6eyJTS0lOIjp7InVybCI6Imh0dHA6Ly90ZXh0dXJlcy5taW5lY3JhZnQubmV0L3RleHR1cmUvYjFkZDRmZTRhNDI5MWRkMGI3ZGI1ODZkODY4N2Y3YjM2ODJlYjY0ZGI5MjFkN2U2MjRlZmFhY2M2ZWEyYjY2MyJ9fX0==";

        // 1. Check the cache first to see if the skull already exists
        ItemStack skull = SkullCache.get(texture);

        // 2. If the skull is not in the cache (or has expired), create a new one
        if (skull == null) {
            skull = SkullBuilder.create()
                    .texture(texture)
                    .lore("A cool custom head!", "Generated by HeadzAPI")
                    .build();

            // 3. Put the newly created skull into the cache for future use
            SkullCache.put(texture, skull);
        }

        // 4. Give the skull item to the player
        player.getInventory().addItem(skull);
    }
}
```

## Setup for Developers

To use HeadzAPI in your plugin, follow these steps:

1.  **Add as a Dependency:** Place the `HeadzAPI.jar` file into your server's `/plugins` folder.

2.  **Update your plugin.yml:** Add `HeadzAPI` to the `depend` list in your plugin's `plugin.yml` file. This ensures your plugin loads after HeadzAPI.

    ```yaml
    name: MyAwesomePlugin
    version: 1.0
    main: com.example.myplugin.MyAwesomePlugin
    api-version: 1.21
    depend: [HeadzAPI]
    ```

3.  **Use the API:** You can now access the `SkullBuilder` and `SkullCache` classes from your plugin's code as shown in the usage example above.
